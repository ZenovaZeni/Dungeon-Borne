# Prototype 0.1.2 Notes

Last updated: 2026-05-01

## Purpose

Prototype 0.1.2 is the death and loot-drop clarity checkpoint after the verified Prototype 0.1.1 readability pass.

It does not add new gameplay systems. It only makes enemy death and legendary loot drops easier to read.

## Verified In Play Mode

- Enemies still take damage.
- Enemies still die.
- Enemy death now shows a short flash, shrink, and puff before removal.
- Brute still drops Echo Axe.
- Echo Axe drop now shows a temporary gold beam, ring, and pulse.
- Echo Axe can still be picked up.
- Echo Axe still modifies Cleave into a shockwave.
- Combat loop remains playable.

## Added Placeholder Feedback

- Enemy death flash.
- Enemy death shrink/fade.
- Small primitive death puffs.
- Echo Axe drop beam.
- Echo Axe drop ring.
- Echo Axe spawn pulse.

## Accepted Placeholder Quality

These effects are temporary:

- Enemy death still does not use animation.
- Enemy death does not yet look like falling over.
- Loot beam/ring are primitive debug shapes.
- Audio was not part of Prototype 0.1.2; later passes added placeholder generated audio.
- No final VFX.

## Player Preference

Later death polish should make enemies fall over or collapse rather than fade/shrink away.

That should wait until the project has simple model/animation support or a better placeholder animation plan.

## Next Smallest Step

Prototype 0.1.3 should focus on camera and framing only:

- Keep player and enemies easier to read during combat.
- Make the arena framing feel better.
- Reduce UI overlap if needed.
- Do not add new mechanics.
- Do not add new enemies.
- Do not import art assets yet.
