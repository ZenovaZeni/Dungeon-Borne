# Scope and Roadmap

## Prototype 0.1: Combat Sandbox

Goal: Prove the combat is fun.

Must include:
- Unity project
- low-poly test room
- mobile-style joystick controls
- keyboard/controller-ready input mappings
- one playable Fighter
- basic slash
- dash
- cleave skill
- stomp/shield-break skill
- one ultimate
- three enemy types
- health/damage/death
- cooldown UI
- damage numbers
- simple loot drop

Do not include:
- full story
- full inventory
- multiplayer
- open world
- ten classes
- monetization

## Prototype 0.2: Loot + Leveling

Goal: Prove progression is fun.

Add:
- XP
- levels
- basic stats
- item pickups
- equipment slots
- simple inventory screen
- one legendary modifier item
- first class evolution at level 5

## Prototype 0.3: Dungeon Run

Goal: Prove the repeatable loop.

Add:
- 5 modular rooms
- room transitions
- treasure room
- elite room
- mini boss
- run completion screen
- town return

## Prototype 0.4: Class Identity

Goal: Prove class branching.

Add:
- Fighter -> Guardian / Berserker
- Rogue base class
- Mage base class
- 2 evolutions per base class if possible
- skill unlock UI

## Prototype 0.5: Demo

Goal: A playable demo for feedback.

Add:
- title screen
- settings
- save/load
- intro story scene
- first dungeon chapter
- one boss
- 25+ items
- sound pass
- VFX pass
- performance test on Android

## Never Build Before Combat Feels Good

Do not add more content until:
- movement feels good
- dash feels good
- hit reactions feel good
- enemy attacks are readable
- loot pickup feels satisfying
- skills are easy to use on phone
