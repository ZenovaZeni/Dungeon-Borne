# Game Pillars

## One-Line Pitch

A mobile-first low-poly action RPG where your class evolves through branching paths and legendary loot changes how your skills work.

## Core Fantasy

The player is a rising hero exploring dangerous dungeon chapters tied to a cursed town, lost orders, ancient relics, and evolving class powers.

## Core Player Promise

Descend. Loot. Evolve. Break your build. Go deeper.

## Five Design Pillars

### 1. Combat Must Feel Good First

The player should enjoy moving, dashing, attacking, casting, dodging, and killing enemies before we add story, expansions, PvP, or extra systems.

Combat feel depends on:
- responsive controls
- readable enemy attacks
- hit impact
- sound effects
- VFX
- short cooldown decisions
- clear damage feedback
- meaningful dodge timing

### 2. Loot Should Change Gameplay

Avoid boring loot that only increases numbers.

Good loot changes how skills behave.

Examples:
- Cleave sends a shockwave.
- Dash leaves a shadow clone.
- Fireball splits into smaller bolts.
- Shield Block releases a holy pulse.
- Poison ticks faster against stunned enemies.
- Bleed kills cause fear explosions.

### 3. Class Evolution Is the Soul

Classes should branch into advanced identities.

The emotional payoff:
- I started as a Fighter.
- I became a Berserker.
- My skills changed.
- My gear pushed me toward a build.
- Now I want to make a different character.

### 4. Mobile-First Does Not Mean Shallow

The game should be playable with thumbs:
- left joystick
- right attack button
- dodge button
- 3 active skills
- ultimate button

Depth comes from:
- builds
- cooldown timing
- positioning
- enemy telegraphs
- skill-modifying gear
- class paths

### 5. Expand in Chapters

The game should be expandable through compact, paid or unlockable chapters.

Each chapter can include:
- new town or hub area
- 5-10 dungeon floors
- new enemy faction
- new boss
- new gear sets
- new class relics
- story continuation

No massive open world needed.
